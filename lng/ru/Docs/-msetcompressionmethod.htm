  • fb={NumFastBytes}
  • Установка числа быстрых байтов для Deflate/Deflate64 кодера. Оно может быть в диапазоне от 3 до 258 (257 для Deflate64). Обычно большее число дает немного лучшую степень сжатия и более медленный процесс сжатия. Большие числа параметра быстрых байтов может значительно увеличить степень сжатия в случае, когда файлы содержат длинные идентичные последовательности байтов.
  • pass={NumPasses}
  • Установка числа проходов для Deflate кодера. Оно может быть в диапазоне от 1 до 15 для Deflate и от 1 до 10 для BZip2. Обычно большее число дает немного лучшую степень сжатия и более медленный процесс сжатия.
  • d={Size}[b|k|m]
  • Установка размера словаря для BZip2. Вы должны определить размер в байтах, килобайтах, или Мегабайтах. Максимальное значение для размера словаря - 900000б. Если Вы не определите никакого символа из [b|k|m], то размер словаря будет вычислен как Размер_словаря = 2^Size байт.
  • mt=[off | on | {N}]
  • Установка режима многопоточности. Если Вы имеете многопроцессорную или многоядерную систему, Вы можете получить увеличение скорости, с этим ключом. Эта опция эффективна только для сжатия (с любым методом) и распаковки с BZip2 потоками. Каждый поток в режиме многопоточности использует 32 Мб оперативной памяти для буферизации. Если Вы определяете {N}, HaoZip пробуют использовать N потоков.

GZip использует те же самые параметры как Zip, но GZip сжимает только с методом Deflate. Таким образом GZip поддерживает только следующие параметры: x, fb, pass.

Ограничение размера solid блока обычно уменьшает степень сжатия, но дает следующие преимущества:

    • Уменьшение потерь в случае будущего разрушения архива.
    • Уменьшение времени извлечения группы файлов (или только одного файла), если группа не содержит весь архив.

Обновление solid .7z архивов может быть медленным, так как, может потребоваться некоторое пересжатие.

Пример:

s=100f10m

устанавливает solid режим с ограничением по 100 файлов и по 10 Мб в одном solid блоке.

  • f=[off | on]
  • Включает или отключает фильтры сжатия для исполняемых файлов: dll, exe, ocx, sfx, sys. Это использует BCJ2 фильтр в режиме Ultra и BCJ фильтр в других режимах. По умолчанию режим f=on.
  • hc=[off | on]
  • Включает или отключает сжатие заголовка архива. По умолчанию режим hc=on. Если сжатие заголовка архива будет включено, то некоторые части заголовка архива будут сжаты с LZMA методом.
  • he=[off | on]
  • Включает или отключает кодирование заголовка архива. По умолчанию режим he=off.
  • {N}
  • Устанавливает порядок методов. Это используется также, чтобы связать параметры с методами. Номера должны начаться от 0. Метод, который имеет меньший номер, будет использоваться перед другими.
  • b{C1}[s{S1}]:{C2}[s{S2}]
  • Связывает выходной поток S1 в кодере C1 с входным потоком S2 в кодере C2. Если номер потока не будет определен, то будет использоваться поток с номером 0.

Обычно кодер имеет один входной поток и один выходной поток. В 7z некоторые кодеры могут иметь множественные потоки ввода и вывода.

Например, BCJ2 кодер имеет один входной поток и четыре выходных потока.

  • mt=[off | on | {N}]
  • Устанавливает режим многопоточности. Если Вы имеете многопроцессорную систему или многоядерную систему, Вы можете получить увеличение скорости, с этим ключом. HaoZip поддерживает режим многопоточности только для сжатия LZMA/LZMA2 и сжатия/распаковки BZip2. Если Вы определяете {N}, например mt=4, HaoZip пробуют использовать 4 потока. Сжатия LZMA использует только 2 потока.
  • {N}={MethodID}[:param1][:param2] ... [:paramN]
  • Устанавливает метод сжатия. Вы можете использовать любое число методов. По умолчанию метод LZMA.

Параметры должны быть в одной из следующих форм:

    • {ParamName}={ParamValue}.
    • {ParamName}{ParamValue}, если {ParamValue} - число, а {ParamName}, не содержит чисел.

LZMA

LZMA - алгоритм, основанный на Lempel-Ziv алгоритме. Он обеспечивает очень быструю распаковку (приблизительно в 10-20 раз быстрее, чем сжатие). Требуемые объемы и конфигурации памяти для сжатия и распаковки, также различны (для подробностей см. описание ключа d={Size}[b|k|m]).

    • a=[0|1]
    • Устанавливает режим сжатия: 0 = быстрое, 1 = нормальное. По умолчанию 1.
    • d={Size}[b|k|m]
    • Устанавливает размер словаря для LZMA. Вы должны определить размер в байтах, килобайтах, или Мегабайтах. Максимальное значение для размера словаря - 1 Гб = 2^30 байтов. Значение по умолчанию для LZMA - 24 (16 Мб) в нормальном режиме, 25 (32 Мб) в максимальном режиме (-mx=7) и 26 (64 Мб) в ультра режиме (-mx=9). Если Вы не определите никакой символ для установки [b|k|m], то размер словаря будет вычислен как Размер_словаря = 2^Size байтов. Для того чтобы распаковать файл, сжатый LZMA методом со словарем размером N, Вы нуждаетесь в N байтах доступной памяти (оперативная память).
    • mf={MF_ID}
    • Устанавливает средство поиска соответствия для LZMA. Метод по умолчанию - bt4. Алгоритмы из hc* группы не обеспечивают хорошую степень сжатия, но они часто работают довольно быстро в комбинации с быстрым способом (a=0). Требуемые объемы и конфигурации памяти зависят от размера словаря (параметр "d" в таблице ниже).

Примечание: ваша операционная система также нуждается в некотором количестве физической памяти для внутренних целей. Так что сохраните, по крайней мере, 32 Мб физической памяти неиспользуемыми.

    • fb={N}
    • Устанавливает число быстрых байтов для LZMA. Оно может быть в диапазоне от 5 до 273. По умолчанию - 32 для нормального режима и 64 для максимального и ультра режимов. Обычно большее число дает немного лучшую степень сжатия и более медленный процесс сжатия.
    • mc={N}
    • Устанавливает число циклов (проходов) для средства поиска соответствия. Оно может быть в диапазоне от 0 до 1000000000. Значение по умолчанию - (16 + число_быстрых_байтов / 2) для средств поиска соответствия BT* и (8 + число_быстрых_байтов / 4) для средства поиска соответствия HC4. Если Вы определите mc=0, то LZMA будет использовать значение по умолчанию. Обычно большое число дает немного лучшую степень сжатия и более медленный процесс сжатия. Например, mf=HC4 и mc=10000 могут обеспечить почти ту же самую степень сжатия как mf=BT4.
    • lc={N}
    • Устанавливает число буквенных контекстных битов (старшие разряды предыдущей буквы). Оно может быть в диапазоне от 0 до 8. По умолчанию - 3. Иногда lc=4 дает выгоду для больших файлов.
    • lp={N}
    • Устанавливает число буквенных позиционных битов (младшие биты текущей позиции для букв). Оно может быть в диапазоне от 0 до 4. По умолчанию - 0. Ключ lp предназначен для периодических данных, когда период равен 2^value (где lp=value). Например, для 32-битовых (4-байтовых) периодических данных Вы можете использовать lp=2. Часто лучше установить lc=0, если Вы изменяете ключ lp.
    • pb={N}
    • Устанавливает число позиционных битов (младшие биты текущей позиции). Оно может быть в диапазоне от 0 до 4. По умолчанию - 2. Ключ pb предназначен для периодических данных, когда период равен 2^value (где lp=value).

LZMA2

LZMA2 - модифицированная версия LZMA. Она обеспечивает следующие преимущества перед LZMA:

    • Лучшая степень сжатия для данных которые не могут быть сжаты. LZMA2 может сохранить такие блоки данных в несжатой форме. Также такие данные будут быстрее распаковываться.
    • Лучше поддержка многопоточности. Если Вы сжимаете большой файл, LZMA2 может разбить этот файл на куски и сжать эти куски в несколько потоков.

Параметр

Значение по умолчанию

Описание

c={Size}[b|k|m]

dictSize * 4

Устанавливает размер куска

Если Вы не определяете размер куска, LZMA2 устанавливает его в значение DictionarySize * 4.

LZMA2 также поддерживает все параметры LZMA, но lp+lc не может быть более чем 4.

Использование LZMA2: 1 поток для каждого куска в режимах x1 и x3; и 2 потока для каждого куска в режимах x5, x7 и x9. Если LZMA2 собирается использовать только такое число потоков, требуемых для одного куска, он не разбивает поток по кускам. Таким образом Вы можете получить различную степень сжатия для различного числа потоков. Вы можете получить лучшую степень сжатия, когда Вы используете 1 или 2 потока.

PPMd

PPMd - алгоритм, базируемый на PPM. Этот алгоритм главным образом основан на исходном коде PPMdH Дмитрия Шкарина. PPMd обеспечивает очень хорошую степень сжатия для простых текстовых файлов. Нет никакого различия между скоростью сжатия и скоростью распаковки. Требуемые объемы и конфигурации памяти для сжатия и распаковки также одинаковы.

Устанавливает порядок моделей для PPMd.
    • mem={Size}[b|k|m]
    • Устанавливает размер используемой памяти для PPMd. Вы должны определить размер в байтах, килобайтах, или Мегабайтах. Максимальное значение - 2Гб = 2^31 байтов. По умолчанию - 24 (16 Мб). Если Вы не будете определять никакого символа для установки [b|k|m], то размер памяти будет вычислен как (2^Size) байтов. PPMd использует одинаковый объем памяти для сжатия и распаковки.
    • o={Size}
    • Устанавливает порядок моделей для PPMd. Размер должен быть в диапазоне [2,32]. По умолчанию - 6.

BCJ2

BCJ2 - конвертер переходов для 32-разрядного x86 исполняемых программ (версия 2). Он преобразовывает некоторые команды перехода для того, чтобы увеличить дальнейшее сжатие.

BCJ2 кодер имеет один входной поток и четыре выходных потока:

    • s0: основной поток. Он требует дальнейшего сжатия.
    • s1: поток для преобразованных значений CALL. Он требует дальнейшего сжатия.
    • s2: поток для преобразованных значений JUMP. Он требует дальнейшего сжатия.
    • s3: сервисный поток. Он уже сжат.

Если используется LZMA, размер словаря для потоков s1 и s2 может быть намного меньшим (512 кб - достаточно для большинства случаев) чем размер словаря для потока s0.

Delta

Это позволяет установить дельта смещение в байтах. Например, чтобы сжать 16-битовые стерео WAV файлы, Вы можете установить "0=Delta:4". По умолчанию дельта смещение равно 1.

XZ

XZ теперь поддерживает только LZMA2 кодер. Ключи аналогичны ключам для 7z формата.

Примеры

HaoZipC a -tzip archive.zip *.jpg -mx0

добавляет *.jpg файлы в архив archive.zip без сжатия.

HaoZipC a -t7z archive.7z *.exe *.dll -m0=BCJ -m1=LZMA:d=21 -ms -mmt

добавляет *.exe и *.dll файлы в solid архив archive.7z используя LZMA метод с 2 Мб словарем и BCJ конвертером. Сжатие будет использовать многопоточную оптимизацию.

HaoZipC a -t7z archive.7z *.exe *.dll -m0=BCJ2 -m1=LZMA:d23 -m2=LZMA:d19 -m3=LZMA:d19
     -mb0:1 -mb0s1:2 -mb0s2:3

добавляет *.exe и *.dll файлы в архив archive.7z, используя BCJ2 конвертер, LZMA со словарем 8 Мб для основного выходного потока (s0), и LZMA со словарем 512 кб для s1 и s2 выходных потоков BCJ2.

HaoZipC a -t7z archive.7z *.txt -m0=PPMd

добавляет *.txt добавляет в архив archive.7z, используя PPMd метод.

Команды, которые могут использоваться с этим ключом

a (Add), d (Delete), u (Update)
